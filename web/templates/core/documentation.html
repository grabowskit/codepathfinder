1. Getting Started

CodePathfinder helps you index and semantically search your GitHub repositories. Once indexed, you can use Claude Desktop or the API to search your codebase using natural language.

What is CodePathfinder?

CodePathfinder is a semantic code search platform that:

  • Creates vector embeddings and indexes from your GitHub repositories for intelligent search
  • Uses AI-powered semantic search to find relevant code by meaning, not just keywords
  • Integrates with Claude Desktop, ChatGPT, and Gemini via MCP (Model Context Protocol)
  • Provides 21 powerful tools for code search, GitHub operations, skills, and job management
  • Supports 17+ programming languages and file formats natively
  • Includes an AI Chat interface powered by LibreChat with multi-model support (GPT-5, Claude Sonnet 4)

Quick Start Guide

  1. Sign Up or Log In
    Create an account or sign in with Google. New accounts may require administrator approval.
  2. Create a Project
    Navigate to Projects in the sidebar and click + New Project.
  3. Configure Your Repository
    Enter your repository details:
    • Name: A friendly name for your project
    • Repository URL: Full HTTPS URL (e.g., https://github.com/org/repo)
    • GitHub Token: Required for private repositories
    • Branch: Specific branch to index (optional)
  4. Run the Indexer
    Click Save & Run to start indexing, or Save to run later.
  5. Start Chatting or Connect Claude Desktop
    Use the built-in AI Chat at Chat in the sidebar, or generate an API key to connect Claude Desktop.

Hint: For private repositories, you'll need a GitHub Personal Access Token with repo scope. Generate one at GitHub Settings → Developer settings → Personal access tokens.

2. Project Management

Projects are the core of CodePathfinder. Each project represents a GitHub repository that you want to index and search.

Creating a Project

Click + New Project from the Projects page and fill in the following fields:

Field Description Required
Name A descriptive name for your project Yes
Repository URL Full HTTPS URL to your GitHub repository Yes
GitHub Token Personal Access Token for private repos For private repos
Branch Specific branch to index (defaults to main/master) No
Clean Index Rebuilds the entire index from scratch No
Pull Before Index Runs git pull before indexing No
Watch Mode Keeps indexer running to watch for changes No

Project Actions

Manage your projects using these essential actions available in the interface:

Icon Action Description
Share

Grant other users read-only access to your project. Shared users can use MCP tools to search the index but cannot modify project settings.
API Keys

Create and manage project-scoped access keys for Claude Desktop, MCP bridges, or direct programmatic API integration.
Edit

Update repository configurations, change branches, or adjust advanced indexer settings like concurrency and watch mode.
Clone

Quickly replicate a project configuration. Useful for A/B testing different indexing strategies on the same repository.
Run

Initiates the containerized indexing job. This will clone the codebase and start building the semantic search index.
Delete

Permanently removes the project and its associated search index. Note: This action cannot be reversed.

Project Status & Visibility

CodePathfinder separates the technical Status of a project from its manual Visibility toggle:

Visibility Toggle (On/Off)

Located next to the project name. Turn On to enable searching via the Chat app or API. Turn Off to hide the project from search tools without losing your index.

Indexing Statuses:

  • Pending - Project created but indexer has not run yet
  • Running - Indexer is currently processing the repository
  • Watching - Initial index complete, monitoring for changes
  • Completed - Indexing complete, project is ready
  • Stopped - Indexing was manually halted
  • Failed - Indexing failed (check console for errors)

Indexer Console

The console at the bottom of the Projects page shows real-time logs from running indexer jobs:

  • Click the Console bar to expand/collapse the log view
  • Logs refresh automatically every 10 seconds
  • Newest logs appear at the top
  • Look for [ERROR] messages if indexing fails

Warning: Large repositories may take several minutes to index. Do not stop the indexer unless necessary, as partial indexes may produce incomplete search results.

3. Skills

Skills are reusable AI agent instructions that enhance code analysis and development workflows. They provide structured prompts and configurations that guide the Chat agent's behavior.

What are Skills?

Skills define how the AI agent should approach specific tasks. Each skill includes:

  • Instructions: Detailed prompts that guide the agent's behavior and output format
  • Allowed Tools: Which MCP tools the skill can use (e.g., semantic_code_search, read_file_from_chunks)
  • Tags: Categories for organizing and filtering skills
  • Context Files: Optional reference files included with the skill

Browsing Skills

Navigate to Skills in the sidebar to view all available skills:

  • Search: Find skills by name or description
  • Filter by Curated: Show only administrator-verified skills
  • Filter by Tag: Click any tag to filter by category
  • Usage Count: See how many times each skill has been used

Creating a Skill

Click + Create Skill to define a new skill:

Field Description Required
Name Unique identifier (e.g., code-review, test-generator) Yes
Description Brief summary shown in the skills list Yes
Instructions Full prompt/instructions for the AI agent (supports Markdown) Yes
Allowed Tools JSON array of tool names, e.g., ["semantic_code_search"] No
Tags JSON array of category tags, e.g., ["python", "testing"] No

Using Skills in Chat

The Chat interface includes a built-in Skill Selector that lets you enhance your conversations with skill-specific instructions.

How to use the Skill Selector:

  1. Top Skills Cards: Your most-used skills appear as clickable cards above the chat input. Click one to select it.
  2. Search for Skills: Type in the search box to find skills by name, description, or tags.
  3. Selected Skill Badge: When a skill is selected, a badge appears showing the active skill. Click X to clear it.
  4. Per-Message Selection: You can change skills between messages - each message can use a different skill.

When you send a message with a skill selected, the skill's instructions are included with your message to guide the AI's response.

Importing Skills

Skills can be imported from SKILL.md files via the UI, API, or MCP tool.

SKILL.md Format

---
name: my-skill-name
description: Brief description of what the skill does

  - read_file_from_chunks
  - github_create_issue

  - code-quality
  - automation

More detailed guidance...

Import Methods

Skills Builder UI Go to Skills Builder → Create Skill and paste your SKILL.md content, or fill in the form fields manually.
API Endpoint POST /skills/import/ with JSON body: {"content": "---\nname: ..."}

4. API & MCP Tool Calling

CodePathfinder provides a REST API and MCP (Model Context Protocol) integration for programmatic access to your indexed code.

Generating API Keys

  1. Navigate to your project in the Projects list
  2. Click the API Keys button (key icon)
  3. Enter a descriptive label (e.g., "Claude Desktop - Home")
  4. Click Generate Key
  5. Copy the key immediately - it will only be shown once!

Important: API keys are only displayed once when created. Store them securely. If you lose a key, you'll need to generate a new one and update your configurations.

Connecting Claude Desktop

There are two ways to connect Claude Desktop to CodePathfinder:

Option 1: OAuth Connector (Recommended)

Connect via OAuth — no API key needed:

  1. Open Claude Desktop → Settings → Connectors
  2. Click Add Connector → Custom
  3. Enter Name: CodePathfinder, URL: https://codepathfinder.com/mcp
  4. Click Add, then authorize with your CodePathfinder account

Option 2: API Key Bridge

For environments where OAuth isn't available, use an API key with the MCP bridge:

  1. Generate an API key for your project
  2. Add this to your Claude Desktop config file (claude_desktop_config.json):
    3. {
  "mcpServers": {
    "codepathfinder": {
      "command": "npx",
      "args": ["-y", "@grabowskit/mcp-bridge"],
      "env": {
        "CODEPATHFINDER_API_KEY": "YOUR_API_KEY",
        "CODEPATHFINDER_API_ENDPOINT": "https://codepathfinder.com/mcp/"
      }
    }
  }
}
  3. Restart Claude Desktop

🟢 ChatGPT (Developer Mode)

ChatGPT supports MCP through Developer Mode (Pro, Plus, Business, Enterprise accounts):

  1. Settings → Apps & Connectors → Advanced settings → Enable Developer mode
  2. Create a new app in the Connectors tab with your CodePathfinder endpoint
  3. Use Developer mode in conversations to access your tools

🔵 Google Gemini

Gemini CLI and Gemini Code Assist support MCP with the same configuration format as Claude Desktop.

🔧 Direct API Access

For direct API access or custom integrations, use project-scoped API keys:

  1. Generate an API key from your project's API Keys page
  2. Use the key in the Authorization: Bearer YOUR_API_KEY header
  3. Make requests to POST https://codepathfinder.com/mcp/ using MCP JSON-RPC format

6. User Management Admin

As an administrator, you have full control over user accounts and access.

Accessing User Management

Navigate to Users in the sidebar to view all registered users.

User Actions

  • View Users: See a list of all users with their email, status, and registration date.
  • Approve Users: New registrations may require approval. Toggle the "Active" checkbox to enable/disable accounts.
  • Edit Users: Modify user details, change passwords, or update permissions.
  • Grant Admin Access: Check "Superuser status" to give a user administrator privileges.
  • Delete Users: Remove user accounts. This will also delete all their projects.

User Roles

Role Capabilities
Standard User Create/manage own projects, generate API keys, view shared projects
Administrator All standard user capabilities + manage all users + view all projects

Warning: Deleting a user will permanently delete all their projects and indexed data. This action cannot be undone.

Configure a GitHub repository to sync AI agent skills from:

Setting Description
Repository URL GitHub repository URL containing skill definitions (e.g., https://github.com/org/skills-repo)

  1. Create a GitHub repository to store your skill definitions

{% if user.is_superuser %}8{% else %}6{% endif %}. Troubleshooting

Common issues and how to resolve them.

Indexer is stuck in "Running" status

This can happen if the indexer job crashes or loses connection.

  1. Click the Stop button to terminate the job
  2. Check the Console for any error messages
  3. Try running the indexer again
  4. If the problem persists, enable "Clean Index" and run again
Private repository indexing fails

Private repositories require a valid GitHub Personal Access Token.

  1. Go to GitHub Settings → Developer settings → Personal access tokens
  2. Generate a new token with repo scope
  3. Edit your project and update the GitHub Token field
  4. Run the indexer again
Claude Desktop can't connect to CodePathfinder

Check the following:

  1. Verify your API key is correct (generate a new one if unsure)
  2. Ensure the config JSON syntax is valid
  3. Restart Claude Desktop after making config changes
  4. Check Claude Desktop logs: ~/Library/Logs/Claude/ (macOS)
Search returns no results

This can happen if:

  • The project hasn't finished indexing (check for "Completed" or "Watching" status)
  • The project is toggled "Off" (check the Visibility switch next to the project name)
  • The search query doesn't match any indexed content
  • The index is empty or corrupted (try "Clean Index" and re-run)
API returns "Invalid API key" error

The API key may be invalid or revoked.

  1. Check that the key starts with cpf_
  2. Verify the key hasn't been revoked in the API Keys page
  3. Generate a new key if the old one is lost or compromised
  4. Make sure you're using the full key, not just the prefix
Rate limit exceeded (429 error)

API requests are limited to 60 per minute per API key.

  • Wait a minute before retrying
  • Reduce the frequency of API calls
  • Contact support for higher rate limits if needed
Console shows "[ERROR]" messages

Common error causes:

  • Authentication failed: Check GitHub Token for private repos
  • Repository not found: Verify the repository URL is correct
  • Branch not found: Check that the specified branch exists
  • Elasticsearch error: Contact support if this persists
