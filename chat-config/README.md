# LibreChat Configuration

This directory contains LibreChat configuration files for the embedded chat panel.

## Files Overview

### Base Configuration (OSS - Committed to Repo)

- **`librechat.base.yaml`** - Base configuration with only CodePathfinder MCP server
  - ✅ Committed to OSS repository
  - Contains minimal MCP setup for public distribution
  - Used as reference for creating local/private configs

### Private Configuration (Not Committed)

- **`librechat.local.yaml`** - Local development configuration with additional MCP servers
  - ❌ NOT committed to OSS repository (`.gitignore`)
  - Includes Notion, Linear, and other private MCP servers
  - Used by `docker-compose.yml` for local development

- **`librechat.yaml`** - Legacy file (now superseded)
  - ❌ NOT committed to OSS repository (`.gitignore`)
  - Kept for backward compatibility only

### Environment Variables

- **`.env`** - LibreChat secrets and API keys
  - ❌ NOT committed (`.gitignore`)
  - Generated by `setup.sh` or manually created
  - Contains CREDS_KEY, JWT_SECRET, AWS credentials, etc.

## Setup Instructions

### For OSS Users (Base Setup)

1. Copy the base configuration:
   ```bash
   cp chat-config/librechat.base.yaml chat-config/librechat.local.yaml
   ```

2. Create environment file:
   ```bash
   # Copy from template if available, or create new
   touch chat-config/.env
   ```

3. Configure your LLM provider credentials in `.env` (examples):
   ```env
   # AWS Bedrock (Anthropic Claude)
   BEDROCK_AWS_ACCESS_KEY_ID=your_key
   BEDROCK_AWS_SECRET_ACCESS_KEY=your_secret
   BEDROCK_AWS_DEFAULT_REGION=us-east-2

   # OR OpenAI
   OPENAI_API_KEY=sk-your_openai_key

   # OR Anthropic Direct
   ANTHROPIC_API_KEY=sk-ant-your_anthropic_key

   # OR Google Gemini
   GOOGLE_API_KEY=your_google_key

   # OR Azure OpenAI
   AZURE_API_KEY=your_azure_key
   AZURE_OPENAI_ENDPOINT=https://your-endpoint.openai.azure.com/
   ```

   See LibreChat documentation for full list of supported providers.

4. Start services:
   ```bash
   docker compose up -d
   ```

### For Private Deployments (Extended Setup)

If you want to add Notion, Linear, or other MCP servers:

1. Copy the local configuration (already includes extended MCP):
   ```bash
   # librechat.local.yaml is already configured with Notion/Linear
   # No action needed if file exists
   ```

2. Deploy MCP server containers:
   ```bash
   # MCP servers are defined in docker-compose.yml
   docker compose up -d notion-mcp linear-mcp
   ```

3. Users configure their own API tokens through LibreChat UI

## Configuration Architecture

### Base (OSS)
```
LibreChat → CodePathfinder MCP only
```

### Extended (Private)
```
LibreChat → CodePathfinder MCP
          → Notion MCP (user-provided tokens)
          → Linear MCP (user-provided tokens)
```

## Security Model

### Per-User Authentication
- Each user provides their own API credentials through LibreChat UI
- Tokens are encrypted and stored per-user in MongoDB
- No shared credentials between users
- MCP servers are stateless (no token storage)

### Files Not in OSS Repo
The following files contain private MCP configurations and are excluded from the OSS repo:
- `chat-config/librechat.local.yaml` - Extended MCP config
- `chat-config/librechat.yaml` - Legacy extended config
- `kubernetes/librechat/librechat-configmap.private.yaml` - Production extended config
- `kubernetes/mcp-servers.yaml` - MCP server deployments
- `mcp-servers/` - MCP server Docker images

## Production Deployment

### Base Production (OSS)
```bash
# Use base configmap (only CodePathfinder MCP)
kubectl apply -f kubernetes/librechat/librechat-configmap.yaml
```

### Extended Production (Private)
```bash
# Deploy MCP servers first
kubectl apply -f kubernetes/mcp-servers.yaml

# Use private configmap (includes Notion/Linear)
kubectl apply -f kubernetes/librechat/librechat-configmap.private.yaml
```

## Updating Configuration

### Local Development
1. Edit `chat-config/librechat.local.yaml`
2. Restart LibreChat: `docker compose restart librechat`

### Production
1. Edit `kubernetes/librechat/librechat-configmap.private.yaml` (or `.yaml` for base)
2. Apply: `kubectl apply -f kubernetes/librechat/librechat-configmap.private.yaml`
3. Restart: `kubectl rollout restart deployment librechat -n codepathfinder`

## Troubleshooting

### LibreChat doesn't see MCP servers
- Check that `librechat.local.yaml` exists
- Verify docker-compose mounts the correct file
- Restart: `docker compose restart librechat`

### MCP connection fails
- Check MCP server containers are running: `docker ps | grep mcp`
- Verify network connectivity: `docker compose exec librechat ping notion-mcp`
- Check LibreChat logs: `docker compose logs librechat | grep -i mcp`

### OSS users want MCP functionality
- Base config includes CodePathfinder MCP (always available)
- Additional MCP servers require private deployment
- Users can self-host Notion/Linear MCP servers using the architecture docs
